  • 富鈦居中協調和要求事項
    1. 鼎捷需先在富鈦局網內，提供數據庫訪問參數和帳密，使用唯讀(READ ONLY)視圖(VIEW)，欄位名稱和中文說明如下︰
      • HR001_DEPT 部門編碼對照表 ︰
        • DEPT_ID 部門編碼
        • DEPT_NAME 部門
      • HR001_JOB 職稱編碼對照表
        • JOB_ID 職稱編碼
        • JOB_NAME 職稱
      • HR001_LEVEL 審核層級編碼對照表
        • LEVEL_ID 審核層級編碼
        • LEVEL_NAME 審核層級
      • 如同同鑫的中間表，規格詳如後述。視圖名稱為HR001_EMPE。
        • 初次提供時，HIRE_DATE 到廠日期，LEAVE_DATE 離職日期，兩欄位均為空。
        • 等第一次按同鑫中間表更新後，HIRE_DATE 到廠日期，LEAVE_DATE 離職日期，按同步後顯示。
    2. 同鑫需在鼎捷提供基礎對照表後，調整目前HR系統，或新增映對的方式，方便富鈦人事部門人員以新核定的部門職稱等，來操作和維護。
    3. 同鑫所提供的中間表規格如下，所有欄位均為文本格式，到廠日期和離職日期使用ISO 8601日期格式，例如2017-05-03，欄位名稱和中文說明如後︰
      • EMPE_ID 員工工號 (PRIMARY KEY 欄位)
        • 只限正式員工工號和試用員工工號，不含臨時工工號。
        • 試用員工轉正後工號不得變更。
      • EMPE_LASTNAME 員工姓
      • EMPE_FIRSTNAME 員工名
      • DEPT_ID 部門編碼
      • DEPT_NAME 部門
      • JOB_ID 職稱編碼
      • JOB_NAME 職稱
      • LEVEL_ID 審核層級編碼
      • LEVEL_NAME 審核層級
      • SUP_ID 直屬主管工號
      • SUP_NAME 直屬主管姓名
      • HIRE_DATE 到廠日期
      • LEAVE_DATE 離職日期︰
        • 預計2017-06-01本項目開始正式運行，凡離職日期在2017-06-01之前的員工信息不出現在本中間表。
        • 本項目正常運行六個月後再行討論是否設定離職日期超過某天數後不在中間表顯示。
    4. 鼎捷仍需提供如同同鑫的中間表，方便三方驗證。
      • 請確認並標示，HIRE_DATE 到廠日期，LEAVE_DATE 離職日期，兩個欄位所在的表單。因為在員工的運行程序裡，我們看不到相關信息。
